# GAP: No customer discovery conducted

**What**: The entire project has been built assuming the problem is "solar farm operators need yield data." Zero interviews with actual solar farm operators, EPCs, or bank financing teams.

**Why it matters**: This is the highest-risk assumption in the project. Every feature built for the wrong problem is wasted effort. The most common failure mode for technical solar products is building for engineers' intuition of the problem rather than operators' actual pain.

**What is needed**: 3–5 customer discovery interviews before any further feature development.

**Risk level**: CRITICAL — blocks all further development
