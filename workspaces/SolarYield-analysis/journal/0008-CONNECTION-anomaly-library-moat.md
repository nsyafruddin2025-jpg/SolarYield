# CONNECTION: The anomaly library IS the data moat

**What**: Across data quality, ML architecture, sustainability, and competitive edge, one theme keeps recurring: the anomaly pattern library is the most valuable and most defensible asset.

**Why it connects everything**:

- **Data quality**: Systematic anomaly tagging requires domain expertise — not just automated threshold detection
- **ML architecture**: The residual hybrid gives clean anomaly signals; the anomaly library makes those signals interpretable
- **Sustainability**: The anomaly library compounds over time — 100 site-years of tagged anomalies is irreplaceable
- **Competitive edge**: A new entrant needs 3–5 years and 50+ sites to build an equivalent library

**The operational process that builds it**:

1. Automated detection: residual < -2σ → flag as anomalous
2. Human triage: operator reviews → what caused this?
3. Tag taxonomy: haze / soiling / inverter_fault / shading / weather_forecast_error / sensor_error / unknown
4. Feedback loop: confirmed tags → improve detection precision

**What to do next**: Start tagging anomalies NOW, even with synthetic/manual labels. The library compounds faster than most founders expect. After 20 site-years of tagged data, it becomes a genuine competitive moat.
